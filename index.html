Conceitos Fundamentais

Controle de Versão

Imagine que você está escrevendo um livro. A cada dia, você salva uma nova versão do arquivo (`livro_v1.doc`, `livro_v2.doc`, `livro_final.doc`). O controle de versão automatiza isso de forma inteligente. Ele permite que você salve "fotos" (snapshots) do seu projeto em momentos específicos, veja o que mudou entre uma foto e outra, e até mesmo volte para uma foto antiga se algo der errado.

Git

É o software de controle de versão que você instala no seu computador. Ele é a ferramenta que tira essas "fotos" (chamadas de commits) e gerencia todo o histórico do seu projeto localmente.

GitHub

É uma plataforma online (um site) que hospeda seus projetos Git. Pense nele como uma rede social para códigos. Ele permite que você guarde uma cópia de segurança do seu projeto na nuvem, colabore com outras pessoas e mostre seu trabalho para o mundo.

Analogia: O Git é como o Microsoft Word, que você usa para escrever no seu computador. O GitHub é como o Google Drive, onde você salva o trabalho online para compartilhar e colaborar.

Prepare seu Ambiente

1. Instale o Git

Acesse o site oficial do Git e baixe a versão para o seu sistema operacional. Siga as instruções de instalação padrão.

2. Crie uma Conta no GitHub

Acesse github.com e crie uma conta gratuita. É rápido e simples.

3. Configure o Git no seu Computador

Abra o terminal (ou Git Bash no Windows) e digite os seguintes comandos, substituindo pelas suas informações. Isso é importante para que o Git saiba quem é o autor dos commits.

git config --global user.name "Seu Nome"
git config --global user.email "seu.email@exemplo.com"

O Fluxo de Trabalho Básico

Este é o ciclo que você mais usará no seu dia a dia. Ele acontece em três etapas principais: modificar seus arquivos, adicionar as mudanças para a "área de preparação" (staging area) e, finalmente, "commitar" (salvar) essas mudanças no seu histórico local.

Iniciar um Repositório (`git init`)

Dentro da sua pasta de projeto, no terminal, digite:

git init

Adicionar Arquivos (`git add`)

Para preparar um arquivo específico para o commit:

git add nome-do-arquivo.txt

Para adicionar todos os arquivos modificados na pasta:

git add .

Salvar as Mudanças (`git commit`)

Salve a "foto" do seu projeto com uma mensagem descritiva:

git commit -m "Mensagem descritiva sobre a mudança"

Comandos Essenciais

git status

Mostra o estado do seu diretório de trabalho e da área de preparação. Use-o constantemente para ver quais arquivos foram modificados e o que está pronto para ser commitado.

git log

Exibe o histórico de commits, mostrando o autor, data e mensagem de cada commit. Essencial para entender a evolução do projeto.

git push

Envia seus commits locais para o repositório remoto (como o GitHub). É como fazer o upload das suas mudanças para a nuvem.

git pull

Busca e mescla as mudanças do repositório remoto para o seu diretório de trabalho local. Use antes de começar a trabalhar para garantir que você tem a versão mais recente.

git clone [url]

Cria uma cópia local de um repositório que já existe remotamente. É o primeiro passo para começar a trabalhar em um projeto existente.

git branch [nome]

Cria uma nova "ramificação" (branch), uma linha de desenvolvimento independente. Permite trabalhar em novas funcionalidades sem afetar a linha principal (`main`).

git checkout [nome]

Muda para a branch especificada, permitindo que você trabalhe nela.

git merge [nome]

Junta o histórico da branch especificada na sua branch atual. É como você integra uma nova funcionalidade de volta à linha principal.
